In Nepal , we have been developing a tool called OSMSG to track the user level mapping statistics to help us organize our events , mapping activities and monitor , In this talk we will talk about the tool we developed , how we developed it ! This talk involves our architecture with duckdb , postgresql , API's and standalone python processor built on top of osmium ! This tool doesn't do anything else rather than providing the OSM mapping statistics in user level ! Similar to how leaderboard used to work  ! It would be our official public launch of the tool to wider OSM community !
